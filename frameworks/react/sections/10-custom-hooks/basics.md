# 自定义 Hooks（通俗讲解）

## 1) 建立直觉
抽离可复用状态逻辑，封装 useToggle/useLocalStorage/useDebounce。

你可以把 React 看成：
- 先描述“当前状态下 UI 长什么样”；
- 当状态变化时，React 重新计算并更新界面。

本节建议先跑通 example，再做 exercise。通过“看得见的变化”反推概念，会更稳。

## 2) 常见误区
- 误区 1：只记语法，不理解数据流。
  - 修正：先画出数据从哪里来、往哪里去。
- 误区 2：一上来追求复杂抽象。
  - 修正：先把一个小场景写通，再提炼复用。
- 误区 3：忽略边界状态（空数据、加载中、错误态）。
  - 修正：至少准备一个“空态”分支。

## 3) 与 Next.js 的衔接
- Next.js 项目可把 hooks 放在 lib/hooks 中跨页面复用。
- 如果出现浏览器 API（如 `localStorage`、`focus`），记得放在 Client Component。
- 组件设计（props、状态归属、事件回调）在 React 与 Next.js 中是同一套方法。

## 4) 学完后你应能做到
- 能用自己的话解释本节核心概念。
- 能从 example 迁移写法到实际页面。
- 能定位并修复本节典型错误。
